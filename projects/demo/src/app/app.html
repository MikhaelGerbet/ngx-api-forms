ngx-api-forms

Bridge API validation errors to Angular Reactive Forms, automatically.

Why ngx-api-forms?

API to Forms Bridge

Automatically maps backend validation errors to Angular Reactive Forms controls. No more manual string matching.

Multi-Backend Presets

Built-in support for NestJS/class-validator, Laravel, Django REST, and Zod. Extensible for any backend.

i18n Ready

Use translation keys or custom resolvers for error messages. Works with any i18n library.

Signals Support

Built with Angular Signals for reactive error state. errorsSignal, firstErrorSignal, hasErrorsSignal.

Submit and Loading State

The handleSubmit() method wraps an Observable, disables the form during the call, and applies API errors automatically on failure.

Implementation

component.ts 
onSubmit() {
  // handleSubmit() does everything:
  //  - disables the form
  //  - sets isSubmittingSignal to true
  //  - on success: re-enables the form
  //  - on error: re-enables + applies errors
  this.bridge.handleSubmit(
    this.http.post('/api/save', this.form.value)
  ).subscribe({
    next: (result) => this.router.navigate(['/success']),
    error: () => { /* errors already applied to form */ },
  });
}
template.html

Internationalization (i18n)

API errors often come in English from the backend. The library supports two approaches for translating them.

i18n Form

How it works

@if (i18nMode() === 'prefix') {

With i18n.prefix, error messages become translation keys:

"email must be valid" becomes "validation.email.isEmail"

Feed these keys to your i18n library (ngx-translate, transloco, etc).

} @else {

With i18n.resolver, you provide a callback that returns the translated message:

"email.isEmail" resolves to "L'email n'est pas valide"

Full control over the translation logic.

} @if (i18nResult()) {

Resolved Errors

{{ i18nResult() }}
}

Implementation

@if (i18nMode() === 'prefix') {
Prefix mode 
bridge = createFormBridge(this.form, {
  preset: classValidatorPreset(),
  i18n: { prefix: 'validation' },
});

// Error messages become keys like:
//   "validation.email.isEmail"
//   "validation.name.isNotEmpty"
// Use with your i18n pipe or service
} @else {
Resolver mode 
bridge = createFormBridge(this.form, {
  preset: classValidatorPreset(),
  i18n: {
    resolver: (field, constraint, originalMessage) => {
      // Use any i18n service here
      return this.translate.instant(`errors.${field}.${constraint}`);
    },
  },
});
}

Dirty State & Interceptors

The isDirtySignal updates reactively when form values change. Interceptors can filter or transform errors before they reach the form, and return a dispose function.

Dirty Tracking

State

isDirty: {{ dirtyBridge.isDirtySignal() }}
hasErrors: {{ dirtyBridge.hasErrorsSignal() }}
interceptor active: {{ interceptorActive() }}

isDirtySignal updates automatically on every value change - no manual checkDirty() call needed.

When the interceptor is active, email errors are filtered out. Use the dispose function to remove it.

Implementation

component.ts 
// isDirtySignal is reactive - no manual call needed
template: `{{ bridge.isDirtySignal() }}`

// Interceptors return a dispose function
const dispose = bridge.addInterceptor((errors) =>
  errors.filter(e => e.field !== 'email')
);
// Later: dispose() to remove the interceptor

Quick Start

component.ts
import { createFormBridge, classValidatorPreset } from 'ngx-api-forms';

export class MyComponent {
  form = inject(FormBuilder).group({
    email: ['', Validators.required],
    name: ['', Validators.minLength(3)],
  });

  bridge = createFormBridge(this.form, {
    preset: classValidatorPreset(),
  });

  onSubmit() {
    this.api.save(this.form.value).subscribe({
      error: (err) => this.bridge.applyApiErrors(err.error),
    });
  }
}
template.html
<form [formGroup]="form" (ngSubmit)="onSubmit()">
  <input formControlName="email" />
  <span ngxFormError="email" [form]="form"></span>

  <input formControlName="name" />
  <span ngxFormError="name" [form]="form"></span>

  <button type="submit">Save</button>
</form>
